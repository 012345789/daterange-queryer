README

1. To execute the program, open this 'index.html' file in your browser.
2. Open up the browser's JavaScript console to run commands. Note that because the language is JavaScript, functions are executed with parentheses and semicolons. Eg.

ADD("a", 10, 20);

QUERY(10);

CLEAR();

The calendar object is the object that contains all the methods and data.

calendar.events.startTimes is a sorted array of custom objects called EventStart. It is sorted by its start time.

calendar.events.endTimes is a JavaScript object/associative array with each key being the id of the event and the value being the end time for that event. Since each id is unique, there will not be any overlapping keys.

Runtime and Space analysis:

The ADD function adds a custom object instance called EventStart with the following data stored in it: start and id. These object instances are stored in a sorted array. Every ADD operation will use quicksort to find where in the array the new object is inserted.

(O(nlogn) run time. O(n) space.)

The ADD function will also add the id and end time to an object which can be read by calling calendar.events.endTimes. (O(C) runtime. O(N) space.)

The QUERY function iterates through the sorted start time array finding any ids with a start time lower than the parameter value. Once it reaches the index of the array which has a value greater than the value, it stops the loop. It then compares the ids with the endTimes object and appends the id to a return value if the end time is greater than the parameter value. (O(n) runtime for start time. O(C) runtime for end time. O(N) space for both.)

The CLEAR function just resets the values inside the calendar object. (O(C) runtime and space)
